StarThinker

Open Source Handover

Deploying recipes and solutions using using StarThinker Framework.


Running A JSON Recipe

You received a recipe from Google, here is how to run it in 15 minutes.

Instructions Minutes
file_download Get the open source framework:
git clone https://github.com/google/starthinker
cd starthinker 		1
build Install dependencies:
pip install -r requirements.txt
source setup.sh 		3
cloud Set up cloud project credentials:
Create a cloud project.
Create client credentials ( other ), download json.
Create service credentials, download json. 		10
directions_run Run a the json recipe:
python all/run.py [recipe json file] -p [project_id] -c [client_json] -s [service_json]

Or schedule recipe to run on schedule:
python cron/run.py [path to recipe json folder] -p [project_id] -c [client_json] -s [service_json] 		1


Running A Solution

You want to run a solution template for a client.

Instructions Minutes
file_download Install StarThinker framework:
Same first three steps as Running A Json Recipe 		14
business Create a solution spec:
Copy template: cp solution/test.py [client_name].json
Change values for your client: vi [client_name].json 		1
settings Generate recipe(s) json:
python solution/run.py [path to client_name].json -d [path to generate recipe json(s) in] 		1
directions_run Run a the json recipe(s):
python all/run.py [recipe json file] -p [project_id] -c [client_json] -s [service_json]

Or schedule recipe to run on schedule:
python cron/run.py [path to recipe json folder] -p [project_id] -c [client_json] -s [service_json] 		1


Coding A Recipe

Create a recipe by assembling tested reliable json instructions.

1. Copy the recipe template...
content_paste cp project/sample/say_hello.json project/[ldap]/[project_name].json
vi project/[ldap]/[project_name].json
2. Add task json from the list below to execute tasks.
list Copy and paste each task JSON into the "task":[...] list.
Replace { "field": ... } with actual values.
You can change the "auth" from "user" to "service".
You can add "day" and "hour" elements to a task to execute at specific times.
Look through the sample recipes in /project/* for reference implementations.
3. Run and test your script. You can run it all or parts to help debug.
settings To run the whole script...
python all/run.py project/[ldap]/[project_name].json --verbose
To run a specific task...
python [task]/run.py project/[ldap]/[project_name].json --verbose -i 1
To run on a schedule ...
python cron/run.py project/[ldap]/

IAS BigQuery

TASK: ias_to_bigquery

Move an IAS report to a BigQuery table.

  1. Schedule an IAS report to be sent to {{EMAIL_TOKEN}}.
  2. The report must be from reports@integralads.com.
  3. Optionally, specify a regular expression for the file link or attachment.
  4. It will be moved to the bigquery table you specify.
{
    "ias": {
        "in": {
            "email": {
                "attachment": {
                    "field": {
                        "default": "", 
                        "kind": "string", 
                        "name": "ias_attachment", 
                        "order": 4
                    }
                }, 
                "to": {
                    "constant": "email_token"
                }, 
                "link": {
                    "field": {
                        "default": "", 
                        "kind": "string", 
                        "name": "ias_link", 
                        "order": 3
                    }
                }, 
                "days": 3, 
                "from": "reports@integralads.com"
            }
        }, 
        "auth": "user", 
        "out": {
            "bigquery": {
                "table": {
                    "field": {
                        "default": "", 
                        "kind": "string", 
                        "name": "ias_table", 
                        "order": 7
                    }
                }, 
                "dataset": {
                    "field": {
                        "default": "", 
                        "kind": "string", 
                        "name": "ias_dataset", 
                        "order": 6
                    }
                }
            }
        }
    }

Aggregate multiple DCM networks into one destination.

  1. See API docs for Metrics.
  2. A report will be created for each account '[Report Name] [Account ID] ( StarThinker )'.
  3. Specify either bucket and path or dataset and table.
{
    "dcm_bulk": {
        "datastudio": {
            "field": {
                "default": true, 
                "kind": "boolean", 
                "name": "datastudio", 
                "order": 11
            }
        }, 
        "name": {
            "field": {
                "default": "", 
                "kind": "string", 
                "name": "name", 
                "order": 2
            }
        }, 
        "auth": "user", 
        "accounts": {
            "field": {
                "default": "", 
                "kind": "integer_list", 
                "name": "accounts", 
                "order": 1
            }
        }, 
        "report": {
            "metrics": {
                "field": {
                    "default": [
                        "impressions"
                    ], 
                    "kind": "string_list", 
                    "name": "dcm_metrics", 
                    "order": 5
                }
            }, 
            "relativeDateRange": {
                "field": {
                    "default": "LAST_7_DAYS", 
                    "kind": "choice", 
                    "order": 3, 
                    "name": "range", 
                    "choices": [
                        "LAST_24_MONTHS", 
                        "LAST_30_DAYS", 
                        "LAST_365_DAYS", 
                        "LAST_7_DAYS", 
                        "LAST_90_DAYS", 
                        "MONTH_TO_DATE", 
                        "PREVIOUS_MONTH", 
                        "PREVIOUS_QUARTER", 
                        "PREVIOUS_WEEK", 
                        "PREVIOUS_YEAR", 
                        "QUARTER_TO_DATE", 
                        "TODAY", 
                        "WEEK_TO_DATE", 
                        "YEAR_TO_DATE", 
                        "YESTERDAY"
                    ]
                }
            }, 
            "type": "STANDARD", 
            "dimensions": {
                "field": {
                    "default": [
                        "date", 
                        "platformType", 
                        "creativeType", 
                        "state", 
                        "dmaRegion"
                    ], 
                    "kind": "string_list", 
                    "name": "dcm_dimensions", 
                    "order": 4
                }
            }, 
            "timeout": 0
        }, 
        "delete": {
            "field": {
                "default": false, 
                "kind": "boolean", 
                "name": "delete", 
                "order": 10
            }
        }, 
        "out": {
            "storage": {
                "path": {
                    "field": {
                        "default": "DCM_Report", 
                        "kind": "string", 
                        "name": "path", 
                        "order": 8
                    }
                }, 
                "bucket": {
                    "field": {
                        "default": "", 
                        "kind": "string", 
                        "name": "bucket", 
                        "order": 7
                    }
                }
            }, 
            "bigquery": {
                "table": {
                    "field": {
                        "default": "", 
                        "kind": "string", 
                        "name": "table", 
                        "order": 6
                    }
                }, 
                "dataset": {
                    "field": {
                        "default": "", 
                        "kind": "string", 
                        "name": "dataset", 
                        "order": 5
                    }
                }
            }
        }
    }

Archive

TASK: archive

Wipe old information from a storage bucket.

  1. Specify how many days back to retain data and which buckets and paths to purge.
  2. Everything under a path will be moved to archive or deleted depending on your choice.
{
    "archive": {
        "storage": {
            "path": {
                "field": {
                    "default": "", 
                    "kind": "string", 
                    "name": "archive_path", 
                    "order": 3
                }
            }, 
            "bucket": {
                "field": {
                    "default": "", 
                    "kind": "string", 
                    "name": "archive_bucket", 
                    "order": 2
                }
            }
        }, 
        "days": {
            "field": {
                "default": 7, 
                "kind": "integer", 
                "name": "archive_days", 
                "order": 1
            }
        }, 
        "auth": "service", 
        "delete": {
            "field": {
                "default": false, 
                "kind": "boolean", 
                "name": "archive_delete", 
                "order": 4
            }
        }
    }
}

Bucket

TASK: bucket

Create a bucket on a project.

  1. Specify the name of the bucket and who will have owner permissions.
  2. Existing buckets are preserved.
  3. Adding a permission to the list will update the permissions but removing them will not.
  4. You have to manualy remove grants.
{
    "bucket": {
        "auth": "service", 
        "bucket": {
            "field": {
                "default": "", 
                "kind": "string", 
                "name": "bucket_bucket", 
                "order": 2
            }
        }, 
        "emails": {
            "field": {
                "default": "", 
                "kind": "string_list", 
                "name": "bucket_emails", 
                "order": 3
            }
        }, 
        "groups": {
            "field": {
                "default": "", 
                "kind": "string_list", 
                "name": "bucket_groups", 
                "order": 4
            }
        }
    }
}
