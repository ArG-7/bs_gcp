StarThinker

Open Source Handover

Deploying recipes and solutions using StarThinker Framework.


Running A JSON Recipe

You received a recipe from Google, here is how to run it in 15 minutes.

Instructions Minutes
file_download Get the open source framework:
Request access to the private ( beta ) repository, email starthinker-help@google.com.
git clone https://github.com/google/starthinker
cd starthinker 		1
build Install dependencies:
pip install -r requirements.txt
source setup.sh 		3
cloud Set up cloud project credentials:
Create a cloud project.
Create client credentials ( other ), download json.
Create service credentials, download json. 		10
directions_run Run a json recipe:
python all/run.py [recipe json path] -p [project id] -c [client json path] -s [service json path] -u [user json path] --verbose

Or schedule recipe to run on schedule:
python cron/run.py [path to recipe destination folder] -p [project id] -c [client json path] -s [service json path] -u [user json path] --verbose 		1


Running A Solution

You want to run a solution template for a client.

Instructions Minutes
file_download Install StarThinker framework:
Same first three steps as Running A Json Recipe 		14
business Create a solution spec:
Copy template: cp solution/test.py [client_name].json
Change values for your client: vi [client_name].json 		1
settings Generate recipe(s) json:
python solution/run.py [path to client_name].json -d [path to generate recipe json(s) in] 		1
directions_run Run a the json recipe(s):
python all/run.py [recipe json file] -p [project_id] -c [client_json] -s [service_json]

Or schedule recipe to run on schedule:
python cron/run.py [path to recipe json folder] -p [project_id] -c [client_json] -s [service_json] 		1


Helpful Tips And Tricks

Best practices for finding, using, and building code on top of StarThinker.

1 Look for helper.py to invoke the code from the command line, helpful for testing setup.
2 Look in solutions/*.json and projects/sample/*.json for samples.
3 Look in [task]/script*.json for json definitions.
4 Review arguments in utils/project/__init__.py at least once.
5 Learn to love: find . -name '*.py' | xargs grep 'somestring'
6 Adding --verbose to python all/run.py or python [task]/run.py produces debug.
7 Read the core framework documentation and code that glues things together:
   util/project/__init__.py
   util/data/__init__.py
   util/google_api/__init__.py
   solution/run.py
8 python solution/run.py [json] --directory [dir] creates recipes.
9 Make use of util/data/__init__.py ( get_rows / put_rows ).
10 Use shared links for sheets.
11 First get the data, then operate on it.
12 There are more than 3 ways to do anything by design.


Coding A Recipe

Create a recipe by assembling tested reliable json instructions.

1. Copy the recipe template...
content_paste cp project/sample/say_hello.json project/[ldap]/[project_name].json
vi project/[ldap]/[project_name].json
2. Add task json from the list below to execute tasks.
list Copy and paste each task JSON into the "task":[...] list.
Replace { "field": ... } with actual values.
You can change the "auth" from "user" to "service".
You can add "day" and "hour" elements to a task to execute at specific times.
Look through the sample recipes in /project/* for reference implementations.
3. Run and test your script. You can run it all or parts to help debug.
settings To run the whole script...
python all/run.py project/[ldap]/[project_name].json --verbose
To run a specific task...
python [task]/run.py project/[ldap]/[project_name].json --verbose -i 1
To run on a schedule ...
python cron/run.py project/[ldap]/

IAS BigQuery

TASK: ias_to_bigquery

Import emailed IAS report into a BigQuery table.

  1. The person executing this recipe must be the recipient of the email.
  2. Schedule a IAS report to be sent to {{ EMAIL_TOKEN }}.
  3. Give a regular expression to match the email link or attachment.
  4. The report will overwrite the table specified.
{
    "ias": {
        "in": {
            "email": {
                "attachment": {
                    "field": {
                        "default": "", 
                        "kind": "string", 
                        "name": "ias_attachment", 
                        "order": 4
                    }
                }, 
                "to": {
                    "constant": "email_token"
                }, 
                "link": {
                    "field": {
                        "default": "", 
                        "kind": "string", 
                        "name": "ias_link", 
                        "order": 3
                    }
                }, 
                "days": 3, 
                "from": "reports@integralads.com"
            }
        }, 
        "auth": "user", 
        "out": {
            "bigquery": {
                "table": {
                    "field": {
                        "default": "", 
                        "kind": "string", 
                        "name": "ias_table", 
                        "order": 7
                    }
                }, 
                "dataset": {
                    "field": {
                        "default": "", 
                        "kind": "string", 
                        "name": "ias_dataset", 
                        "order": 6
                    }
                }
            }
        }
    }
}

                "field": {
                    "default": true, 
                    "kind": "boolean",

    }

  1. Reference field values from the DBM API to build a report.
